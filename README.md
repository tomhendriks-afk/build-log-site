# Build Log, `build.ambient-advantage.ai`

Static site for Ambient Advantage's behind-the-scenes Build Log. Self-contained
HTML files, deployed by Cloudflare Pages on every push to `main`.

This site is the *catalogue* counterpart to the chronological feed at
`take.ambient-advantage.ai`: the index page is a clickable architecture
catalogue (one card per system component) rather than a reverse-chrono list,
with a pinned "Start here" hero card pointing to the inaugural architecture
deep-dive.

## Repo layout

```
build-site/
  publish.py                    Local publish script (Python 3 stdlib)
  _template.html                Per-post page template
  _index_template.html          Catalogue index template
  _tag_template.html            Per-tag listing template
  posts.json                    Generated post catalog, DO NOT hand-edit
  index.html                    Generated catalogue homepage, DO NOT hand-edit
  <slug>.html                   Generated post pages
  tag-<slug>.html               Generated per-tag pages
  drafts/
    EXAMPLE.md                  Frontmatter + markdown reference
    *.md                        Your draft posts
  README.md                     This file
```

## Publish a new post

1. Write a markdown draft in `drafts/<slug>.md`. Start with a frontmatter block;
   see `drafts/EXAMPLE.md` for the full reference.
2. Run:
   ```
   python3 publish.py drafts/<slug>.md
   ```
3. Review the diff:
   ```
   git diff
   ```
4. Commit and push (always pull rebase first, the project repos are touched
   from multiple machines):
   ```
   git fetch origin main && git rebase origin/main
   git add .
   git commit -m "post: <slug>"
   git push origin main
   ```
5. Cloudflare Pages auto-deploys within ~1 minute.

## What `publish.py` does

For each draft you publish, the script:

1. Parses the YAML-ish frontmatter and the markdown body.
2. Renders an HTML article page at `<slug>.html`.
3. Inserts/updates the post in `posts.json` (sorted newest-first).
4. Regenerates `index.html` with the pinned hero, the component grid, and the
   recent-posts list.
5. Regenerates a `tag-<slug>.html` page for every unique tag and for every
   known component (including empty pages, handy for upcoming-component
   placeholders).

The script is **idempotent**, running it twice on the same draft updates
rather than duplicates. It does not touch git; review the diff and commit
yourself.

## Tag taxonomy

The script understands four loose axes of tags. None are enforced, but the
recommended split:

- **Components** (the *what*): `cloud-run`, `cloud-build`, `cloud-scheduler`,
  `firestore`, `gcs`, `secret-manager`, `anthropic`, `elevenlabs`, `gemini`,
  `gmail-api`, `github`, `cloudflare-pages`, `beehiiv`. These map to cards on
  the index page (defined in `COMPONENTS` in `publish.py`).
- **Themes** (the *why*): `architecture`, `cost`, `gotchas`, `security`, `ops`,
  `migration`, `decisions`.
- **Method** (the *how*): `claude`, `claude-cowork`, `gemini`, `agents`,
  `agentic-ai`.
- **Stack umbrella**: `google-cloud`.

Add or rename a primary component by editing the `COMPONENTS` list at the top
of `publish.py`. New entries automatically get a card on the index page and an
empty `tag-<slug>.html` placeholder until a post lands tagged with them.

## Hosting

DNS and hosting follow the same pattern as the other Ambient Advantage sites:

- **Domain:** `build.ambient-advantage.ai`, set up as a Cloudflare CNAME to
  the Cloudflare Pages project.
- **GitHub repo:** create a new public repo (suggested name: `build-log-site`)
  and connect it to a new Cloudflare Pages project. Build command: leave empty
  (this is a static site, no framework). Output directory: `/`.
- **Auto-deploy:** every push to `main` triggers a Cloudflare rebuild. No build
  step is required because the HTML is fully generated by `publish.py` before
  it lands in git.

## Visual identity

The Build Log uses a lavender-purple-on-deep-blue palette to differentiate it
from the warm-orange Chiel's Take site, while sharing the same dark base and
brand-blue logo dot for family resemblance:

- `--bg-0: #05070d`, page background (shared across all four sister sites)
- `--text: #e8eefc`, body text
- `--warm: #a78bfa`, primary accent (lavender purple, build-log-specific)
- `--accent: #4ea8ff`, link colour and the logo dot (shared brand blue)

Note: the variable is still named `--warm` for parity with the other site
codebases, even though the value is now a cool purple. Renaming would touch
every line of CSS without changing what's rendered.

Fonts: Space Grotesk for headings and chrome, Inter for body, JetBrains Mono
for code and tag chips.
